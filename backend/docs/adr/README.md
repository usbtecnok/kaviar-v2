# Architecture Decision Records (ADRs)

Este diretório contém as decisões arquiteturais do KAVIAR Backend.

## ADRs Ativos

- [ADR-001: Domínio Territorial e Operacional de Corridas](./ADR-001-dominio-territorial-operacional.md) - **FINAL** ✅

## Formato

Seguimos o formato padrão de ADRs:
- **Status:** PROPOSTO | ACEITO | REJEITADO | SUPERSEDED
- **Contexto:** Por que esta decisão é necessária
- **Decisão:** O que foi decidido
- **Consequências:** Impactos positivos e negativos

## Processo

1. Toda mudança arquitetural significativa requer ADR
2. ADRs são imutáveis após aceitos
3. Mudanças exigem novo ADR que pode superseder anterior
4. ADRs devem ser referenciados em commits e documentação
